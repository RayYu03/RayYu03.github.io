二维码 - 生成纠错码

Posted on 2017-02-26   |  

纠错码字允许二维码阅读器检测和纠正二维码中的错误。本文主要说明如何在数据编码后创建这些纠错码。

纠错表中包含“组1”和“组2”，以及“块数”。这意味着数据码字必须被分割成多达两个组，并且在每个组内，数据码字可以进一步分成块。数据码字被顺序地分解（即从码字1开始，然后从码字2开始，等等）

在表中，注意组1中的块由15个数据码字组成，组2中的块由的16个数据码字组成，具体分为多少个数据码字是主要根据纠错表。还要注意，它们是顺序分解。也就是说，它们的顺序与在被分成块之前的顺序相同。

根据纠错表，对于5-Q码，每个块有18个纠错码字。在该示例中，存在四个块，因此对于总共72个纠错码字，存在四组18个纠错码字。

如纠错表所示，较小的二维码不需要将数据码字分解，因此，对于1-M码，所有16个数据码字将被用作单个块，并且只需要生成10个纠错码字。

在继续5-Q示例之前，有必要解释一下 Reed-Solomon 纠错的基本步骤。

理解多项式长除法

纠错码字将使用称为Reed-Solomon纠错的方法来生成，部分过程是执行多项式长除法。也就是说，用一个多项式除以另一个多项式。本教程假设你了解如何手工进行标准长除法（即用数字而不是多项式）。

多项式长除法比标准长除法稍微复杂一些。为了帮助澄清过程，这里是一个简单的多项式长除法的例子。

此示例展示了 3x2 + x - 1 (被除数) 除以 x + 1（除数）的过程。这些多项式首先被放入一个表格中（见维基百科上关于多项式长除法的文章）

在长除法的每个步骤中，将x + 1乘以某个值，使其第一项与表格底部的多项式相等。

对于第一步，3x2 + x - 1 位于表格的底部，因此将x + 1乘以3x。得到结果3x2 + 3x，其具有与在表格底部的多项式的第一项相同的第一项。

现在从 3x2 + x - 1 中减去3x2 + 3x。得到-2x-1。

再将x + 1乘以一个项，这将生成一个多项式，其第一项与底部多项式的多项式相同。在这种情况下，执行-2（x + 1）得到-2x - 2。

再次减去两个多项式。

横线之上的多项式即为商，而剩下的1就是余数。

多项式长除法的步骤：

  1. 把被除式、除式按某个字母作降幂排列，并把所缺的项用零补齐。
  2. 将分子的第一项除以分母的最高次项（即次数最高的项)，结果写在横线之上。
  3. 将分母乘以刚得到结果（最终商的第一项），乘积写在分子前两项之下（同类项对齐）。
  4. 从分子的相应项中减去刚得到的乘积（消去相等项，把不相等的项结合起来），结果写在下面。然后，将分子的下一项“拿下来”。
  5. 把减得的差当作新的被除式，重复前三步（直到余式为零或余式的次数低于除式的次数时为止．被除式=除式×商式+余式 ）
  6. 重复第四步。直到得出结果。横线之上的多项式即为商，而剩下的数字就是余数。

对于Reed-Solomon纠错所需的多项式长除法在某些方面比该例子更简单，因为不必处理多项式项的指数。

3 理解伽罗瓦域

如上所述，为了生成纠错码字，该过程使用称为Reed-Solomon纠错的方法。
而多项式长除法则用到了伽罗瓦域，其本质上是数字的有限集合，以及创建仍在该集合中的数字的一些数学运算。

二维码标准使用逐位模2算法和逐字节模100011101算法。这意味着使用伽罗瓦域28，换句话说是伽罗瓦域256，有时也写为GF(256)。

GF(256) 中的数字在0到255（包括）的范围内。也可以用八位字节表示的相同数字范围（最大可能的八位字节是11111111，等于255）。

这意味着GF(256)中的所有数学运算将可以表示为八位字节的数字的运算。

4 理解伽罗瓦域算法

如上所述，GF(256) 中的数字在0到255（包括）的范围内。GF(256)中的数学运算本质上是循环的，这意味着如果在结果大于255的数的GF(256)内执行数学运算，则需要使用模运算来获得仍然在伽罗瓦域中的数。

在伽罗瓦域中，负数具有与正数相同的值，因此-n = n。换句话说，总是使用伽罗瓦域算法中的数字的绝对值。

在伽罗瓦域内的加法和减法是相同的事情。伽罗瓦域中的加法和减法通过执行正常的加法或减法，然后模运算来计算。由于我们使用逐位模2算法（如二维码标准中所述），这与执行XOR运算相同。

5 使用逐字节模100011101生成2的幂

GF(256)中的所有数可以表示为2的幂。具体地，GF(256)中的所有数可以表示为2n，其中0≤n≤255。然而，如前所述，GF(256)中的所有数字本身必须在0到255的范围内，因此28对于Galois域似乎太大，因为它等于256。

2从0到8的幂为：

20 = 1
21 = 2
22 = 4
23 = 8
24 = 16
25 = 32
26 = 64
27 = 128
28 = 256

二维码标准中规定使用逐字节模100011101算法（其中100011101是十进制285的二进制数）。当数字为256或更大时，它应与285异或。

例如:

28 = 256 ^ 285 = 29

当继续到29时，不要直接用它的值512和与285异或（结果太大）。由于29 = 28 * 2，请使用在上一步中计算的值28。

即：

29 = 28 × 2 = 29 × 2 = 58

同样：

210 = 29 × 2 = 58 × 2 = 116
211 = 210 × 2 = 116 × 2 = 232

当结果大于或等于256的值时，再次与285进行异或：

212 = 211 × 2 = 464 ^ 285 = 205

6 对数与反对数乘法

因为所有值可以用2n来表示，所以可以用对数和反对数简化GF(256)中的乘法。(slide rule，不限于GF(256)）。
通常情况下（即不仅在Galois Fields中），可以用以下操作将两个数p和q相乘：

b(logbp + logbq)

假设基数为2，则上述操作变为：

2(log2p + log2q)

而log2x操作告诉我们乘以多少次2得到x。换句话说，当2n = x时，它可以确定n。

这种方式提供了一个用于GF(256)中的数字进行乘法操作的快捷方式。当乘以具有相同基数的两个数字（例如22 × 28 ）时，这等效于将两个指数加在一起，如下：

22 × 28 = 2(2 + 8) = 210

由于所有GF(256)值都可以表示为2的幂，所以GF(256)中的所有乘法都可以作为基数为2的指数加法运算。

例如，16 × 32，结果与24 × 25相同，等价于2(4 + 5) = 29。

29的值在前面已经计算过了，因此在逐字节模285的GF(256)中，16 × 32是29，结果等于58。

当指数相加时，如果指数大于或等于256，只需将指数模255。

即：
2170 × 2164 = 2(170+164) = 2334 → 2334 % 255 = 279.

因此，在GF(256)中执行乘法所需要的是生成2的所有幂。
这些值可以在对数-反对数表中找到。该表使用阿尔法符号，其中α= 2。

7 生成多项式

如上所述，纠错码使用多项式长除法。为此，我们需要两个多项式。第一个多项式称为消息多项式(message polynomial)。消息多项式使用数据编码步骤中得到的数据码字作为其系数。

举个栗子：

如果被转换为整数的数据码字是25,218和35，则消息多项式将是25×2 + 218×+ 35。实际中，用于标准二维码的消息多项式比本例长得多。

消息多项式将与生成多项式进行除法操作。生成多项式是通过(x - α0) … (x - αn-1)乘法生成的多项式。其中n是必须生成的纠错码字的数目（参见纠错表）。如前面部分所述，α等于2。

可以以编程方式创建可能需要的所有生成多项式。在此过程中，本文将使用α和本文前面所述的对数-反对数乘法。

二维码标准中附件A中列出了从7-68的生成多项式(具体参见ISO/IEC 18004)。

虽然标准二维码将总是需要每块多于2个纠错码字，但是为了计算方便，该文将解释如何计算用于2个纠错码字的生成多项式。

7.1 2个纠错码字的生成多项式

首先，将(x - α0) 和 (x - α1)相乘。
因为x的系数为1，且α0 = 1，可以写成如下形式：

(α0x - α0) × (α0x - α1)

将第一部分的每个项乘以第二部分的每个项以得到：

(α0x1 × α0x1) + (α0x0 × α0x1) + (α0x1 × α1x0) + (α0x0 × α1x0)

这里可以使用指数加法来执行乘法：

(α(0+0)x(1+1)) + (α(0+0)x(0+1)) + (α(0+1)x(1+0)) + (α(0+1)x(0+0))

结果为：

α0x2 + α0x1 + α1x1 + α1x0

合并同类项：

α0x2 + (α0+α1)x1 + α1x0

因为GF(256)中的加法通过异或执行。使用对数-反对数表换为其对应的整数，然后执行异或运算。

x2 + (1^2)x1 + 2x0

x2 + 3x1 + 2x0

将系数转为α符号，得到：

α0x2 + α25x1 + α1x0

这就是2个纠错码字的生成多项式。

7.2 3个纠错码字的生成多项式

所有其余的生成多项式可以使用每个上一步骤的生成多项式以相同的方式创建。从上一个生成多项式开始，并乘以下一个因子(x - α2)。

(α0x2 + α25x1 + α1x0) * (α0x1 + α2x0)

(α0x2 α0x1) + (α25x1 α0x1) + (α1x0 α0x1) + (α0x2 α2x0) + (α25x1 α2x0) + (α1x0 α2x0)

使用指数加法来执行乘法：

(α(0+0)x(2+1)) + (α(25+0)x(1+1)) + (α(1+0)x(0+1)) + (α(0+2)x(2+0)) + (α(25+2)x(1+0)) + (α(1+2)x(0+0))

得到：

α0x3 + α25x2 + α1x1 + α2x2 + α27x1 + α3x0

合并同类项：

α0x3 + (α25 ^ α2)x2 + (α1 ^ α27)x1 + α3x0

α0x3 + (3 ^ 4)x2 + (2 ^ 12)x1 + α3x0

α0x3 + 7x2 + 14x1 + α3x0

α0x3 + α198x2 + α199x1 + α3x0

这就是3个纠错码字的生成多项式。

7.3 当指数大于或等于256时

当通过将指数加在一起来执行乘法时，有时这可能导致大于或等于256的指数。在这种情况下，在组合类似项之前应用模255。

例如:
α257x4 = α(257 % 255)x4 = α2x4

其他生成多项式

通常情况下，可以创建所有生成多项式g(x)，如下所示：

(g(x) for 2 error correction codewords) (x - α2) = (g(x) for 3 error correction codewords)
(g(x) for 3 error correction codewords) (x - α3) = (g(x) for 4 error correction codewords)
(g(x) for 4 error correction codewords) (x - α4) = (g(x) for 5 error correction codewords)
…
(g(x) for n-1 error correction codewords) (x - αn-1) = (g(x) for n error correction codewords)
