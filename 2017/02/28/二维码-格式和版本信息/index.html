二维码 - 格式和版本信息

Posted on 2017-02-28   |  

创建二维码的最后一步是生成格式和版本信息字符串，然后将它们放在二维码中的正确位置。本文主要说明如何生成格式和版本信息字符串，然后将它们放在二维码中。

1 格式字符串

格式信息字符串对当前二维码中使用的错误校正级别和掩码模式进行编码。由于存在四个可能的纠错级别（L，M，Q和H）和7个可能的掩码模式，因此存在 4×7 = 28 个可能的格式信息串。

下一节将介绍如何生成这些格式字符串。有关28个格式字符串的完整列表，请参阅格式字符串表。

1.1 生成格式信息字符串

格式信息字符串是一个15个bits的信息，每一个bit的位置如下图所示：（注意图中的Dark Module，那是永远出现的）

这15个bits中包括：

  • 5个数据bits：其中，2个bits用于表示使用什么样的纠错级别，3个bits表示使用什么样的掩码模式。
  • 10个纠错bits。主要通过BCH Code来计算。

然后15个bits还要与101010000010010做XOR操作。这样就保证不会因为我们选用了00的纠错级别和000的掩码，从而造成全部为白色，这会增加我们的扫描器的图像识别的困难。

1.2 纠错 Bits

创建格式信息字符串的第一步是获取指定二维码中使用的错误校正级别的两个位。下表显示每个错误校正级别的位序列。

纠错级别 位 整数
L 01 1
M 00 0
Q 11 3
H 10 2

请注意，表中的数字不按0,1,2,3的顺序排列。

1.3 掩码 Bits

对于掩码图案，请参阅掩码图案页面以找到每个图案对应的掩码编号。将数字转换为3位二进制字符串。

1.4 示例

例如，如果我们使用纠错级别L和掩码模式4来生成二维码，则如下创建5位二进制序列：

01 (纠错码指示符)

100 (掩码模式4的二进制)

结果: 01100

1.5 为格式字符串生成纠错 Bits

现在有了前5位的格式字符串，我们使用它来生成10个纠错Bits。

1.5.1 获取生成多项式

当生成格式字符串的纠错码字时，二维码标准使用以下公式生成多项式：

x10 + x8 + x5 + x4 + x2 + x + 1

我们可以通过仅取每个项的系数将其转换为二进制字符串。

x10的系数为1，x9的系数为0，因为x9不存在于多项式中。

根据以上公式得出表示此步骤的生成多项式的二进制字符串是:
10100110111

1.5.2 计算纠错 Bits

下一步是将格式字符串位（上一步的01100）除以生成多项式（上一步的10100110111）。

为此，首先通过将十个0放在格式字符串的右侧创建一个15位字符串，如下所示：

01100 - > 011000000000000

现在从左边删除所有的0：

011000000000000 - > 11000000000000

具体计算步骤：

  1. 在生成多项式的右边填充0，使其与当前格式字符串的长度相同。
  2. 使用当前格式字符串对填充的生成多项式字符串进行XOR。
  3. 从结果的左侧删除0。

循环以上步骤，直到得到的格式字符串为10位或更少位长。因此，在每次循环之前，检查以确保当前格式字符串是11位或更长（11位是生成多项式的长度）。我们当前的字符串为11000000000000，是14位长。
