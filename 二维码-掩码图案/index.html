二维码 - 掩码图案

Posted on 2017-02-28   |  

现在模块已经被放置在矩阵中，必须确定最佳掩模图案。掩模图案根据特定规则改变模块的亮暗。此步骤的目的是修改二维码图案以使二维码扫描器更容易读取。

1 掩码

如果一个模块被掩码覆盖，则亮模块变暗，暗模块变亮。换句话说，就是改变模块的颜色。

2 掩码图案

二维码标准定义了八个可应用于二维码的掩码图案。

例如，掩码图案＃1对二维矩阵中的每两行进行掩码，对于掩码图案＃2，对二维矩阵中的每三列进行掩码。

2.1 八种掩码图案

当编码二维码时，有八个掩码图案，你可以使用它们来更改输出的矩阵。每个掩码图案根据它们在二维矩阵中的坐标改变位(颜色)。掩码图案的目的是修改二维码图案以使二维码扫描器更容易读取。

每个掩码模式使用公式来确定是否改变当前位的颜色。将当前位的坐标放入公式中，如果结果为0，则使用该坐标处的相反位。例如，如果坐标（1,3）的位为1，并且将该坐标放入公式得到0，则将0置于（1,3）。

这里是掩码模式公式的列表。请注意，二维码标准的第51页有一个错误，它显示哪个轴对应于i​​和j，它应该具有对应于行的j和对应于列的j。此外，第51页上的掩码图案7的公式也不正确。下面列出了正确的公式。

掩码序号 公式
0 (i + j) mod 2 == 0
1 (i) mod 2 == 0
2 (j) mod 3 == 0
3 (i + j) mod 3 == 0
4 ( floor(i / 2) + floor(j / 3) ) mod 2 == 0
5 ((i × j) mod 2) + ((i × j) mod 3) == 0
6 ( ((i × j) mod 2) + ((i × j) mod 3) ) mod 2 == 0
7 ( ((i + j) mod 2) + ((i × j) mod 3) ) mod 2 == 0

注：mod是Modulo(模)的缩写，floor表示向上取整，(i, j)表示横坐标和纵坐标。

3 应用于哪些模块?

掩码模式只应用于数据模块和纠错模块。即下面模块不需要使用掩码：

  • 功能模式
    • finder patterns
    • timing patterns
    • separators
    • alignment patterns
  • 格式信息区域

4 确定最佳掩码

在将掩码图案应用于二维码矩阵时，还需要根据二维码标准中定义的四个评估条件给予评分。二维码编码器必须应用所有的八个掩模图案，并且对每一个图案进行评估。

无论是哪种掩模模式，最终分数最低的就是用于最终输出的掩码模式。

4.1 如何评估预留区域

注意，即使掩码仅应用于数据和纠错模块，也需要评估整个矩阵（包括功能模式图案和保留区域）。

为了正常的进行评估，保留区域都被认定为亮模块。

4.2 四种评分规则

四种评分规则可概括如下：

  • 对于每组五个或更多相同颜色在一行（或列）的模块。
  • 对于矩阵中的每个2x2区域的相同颜色的模块。
  • 看起来类似于finder patterns的图案。
  • 一半以上的模块是暗的或亮的。

4.3 评价条件1

对于第一个评估条件，逐个检查每一行。

  • 如果有五个相同颜色的连续模块，则加3分。
  • 如果在前五个之后有更多相同颜色的模块，则为相同颜色的每个附加模块添加1分。
  • 然后，逐个检查每个列，检查相同的条件。
  • 将行列得分相加得到总分。

在该示例中，行得分是92，列得分是88。

因此，条件＃1的总分是92 + 88 = 180。

4.4 评价条件2

对于第二个评估条件，查找相同颜色的至少2x2个模块或更大的区域。 二维码标准说，对于尺寸为m×n的同色块，评分为3×（m-1）×（n-1）。然而，二维码标准没有规定当存在多个分割同色块的方式时如何计算分数。

因此，不需要寻找大于2×2的固体块，简单地对二维码中相同颜色的每2×2块添加3分，确保计算重叠的2×2块即可。

4.6 评价条件4

第四个评估条件基于亮模块与暗模块的比率。

要计算此评估规则，请执行以下步骤：

  1. 计算矩阵中模块的总数
  2. 计算矩阵中暗模块的总数
  3. 计算矩阵中暗模块的百分比：（暗模块的总数 / 模块的总数) * 100%
  4. 确定此百分比数字是5的多少倍和下一个倍数。例如，对于43％，5的倍数是40，并且5的下一个倍数是45。
  5. 从这个5的倍数中减去50，并取结果的绝对值。例如，| 40 - 50 | = | -10 | = 10，| 45-50 | = | -5 | = 5。
  6. 将第五步的结果分别除以五。例如，10/5 = 2 ，5/5 = 1。
  7. 最后，取最小的数字乘以10。本例中，最小的数字是1，所以结果是10。得到条件4的评分。

在下面的图片中，模块的总数为441，暗模块的总数为213。

  • 暗模块的百分比为（213/441）*100≈48.2993%
  • 5的倍数为45，下一个5的倍数为50
  • 减去50，并取每个的绝对值：
    • |45 - 50| = |-5| = 5
    • |50 - 50| = 0
  • 分别除以5：
    • 5/5 = 1
    • 0/5 = 0
  • 取最小的数字乘以10：
    • 0 * 10 = 0

因为，在本例中，条件四的评分为0.

4.7 最终得分

最终得分 = 条件1的总分 + 条件2的总分 + 条件3的总分 + 条件4的总分
